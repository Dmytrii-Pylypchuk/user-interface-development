## Мета роботи

Ознайомитися з принципами розробки мобільних застосунків для ОС Android, вивчити основні компоненти сучасного графічного інтерфейсу користувача відповідно до принципів **Material Design**, а також реалізувати застосунок для перегляду та редагування текстових файлів із підтримкою роботи з файловою системою мобільного пристрою.

---

## Завдання

Необхідно створити мобільний застосунок **«Текстовий редактор»**, який реалізує:

- створення нового текстового файлу;
- відкриття існуючого текстового файлу з пам’яті телефону;
- редагування текстового вмісту;
- збереження змін у файл;
- збереження нового файлу через системний діалог вибору місця;
- контроль незбережених змін при створенні нового файлу, відкритті іншого файлу або виході із застосунку;
- зрозумілий та якісний інтерфейс користувача відповідно до принципів якості UI.

---

## Опис застосунку

Розроблений застосунок є мобільним текстовим редактором для Android, що дозволяє працювати з текстовими файлами у пам’яті пристрою.

### Основні можливості:
- **New** — створення нового документа;
- **Open** — відкриття існуючого `.txt` файлу через системний **File Picker**;
- **Save** — збереження поточного файлу;
- **Save As** — збереження документа під новим іменем або в іншому місці;
- **Exit / Back** — коректний вихід із перевіркою незбережених змін.

---

## Особливості інтерфейсу

Інтерфейс розроблено відповідно до сучасних принципів **Material Design**:

- використано **Top App Bar**;
- додано **Floating Action Button (FAB)** для швидкого збереження;
- реалізовано зручне меню дій;
- використано **Material Alert Dialog** для підтвердження критичних операцій;
- передбачено індикацію стану документа:
  - назва файлу;
  - позначка про наявність незбережених змін;
- забезпечено зручну навігацію та зрозумілу взаємодію з користувачем.

---

## Контроль незбережених змін

Застосунок реалізує перевірку змін у тексті.

Якщо користувач:
- створює новий файл,
- відкриває інший файл,
- натискає кнопку “Назад”,
- або завершує роботу,

то при наявності незбережених змін відображається діалогове вікно з варіантами:

- **Зберегти**
- **Не зберігати**
- **Скасувати**

Це дозволяє уникнути випадкової втрати даних.

## Технології

У проєкті використано:

- **Java**
- **Android Studio**
- **Android SDK**
- **Material Components for Android**
- **ActivityResultLauncher**
- **Storage Access Framework (SAF)**
- **File Picker (ACTION_OPEN_DOCUMENT / ACTION_CREATE_DOCUMENT)**

## Як запустити проєкт
1. Відкрити проєкт
Запустити Android Studio
Обрати Open
Вибрати папку проєкту TextEditor
2. Дочекатися синхронізації Gradle
Після відкриття проєкту дочекатися завершення Gradle Sync
3. Запустити застосунок
Підключити фізичний Android-пристрій або
Запустити емулятор Android (AVD)
Натиснути Run ▶
